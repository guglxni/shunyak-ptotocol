Shunyak Guardrail Execution Flow

Compliance-first workflow with blocked and authorized branches

Blocked Guarantees

  • - Settlement tool is never invoked on invalid consent.
  • - Runtime returns deterministic blocked outcome.
  • - Audit trail records block reason and event type.

Authorized Guarantees

  • - Settlement only occurs after positive compliance check.
  • - Signing happens at execution boundary, not model context.
  • - Response includes transaction metadata and final status.

Control Plane

  • - WorkflowPolicy controls execution order.
  • - DLP and token guardrails protect tool invocation.
  • - On-chain consent state is the source of truth.
